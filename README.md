# Тестовое задание на стажёра Backend в Avito.Tech Осень 2025

## Описание
Сервис назначения ревьюеров для Pull Request'ов с возможностью управления командами. Сервис позволяет создавать PR(ы), переназначать ревьюеров, изменять статус PR(ов), а также задавать статус активности пользователя (на случай, если кто-то уехал в отпуск)
## Запуск
1. Склонируйте репозиторий с помощью `git clone`
2. Откройте терминал и пропишите `docker compose up`

## Особенности реализации
- По умолчанию вшито 13 пользователей и 3 команды для тестирования
- API максимально точно повторяет исходную спецификацию openAPI
- Для правильной (де)сериализации массивов, хранящихся в PostgreSQL (в частности в объектах `Team`, `PullRequest`) использовался тип данных pq.StringArray и собственный метод сериализации для `TeamMembers`.
- Считается, что у каждого пользователя может быть не более 1 ожидающего ревью. После того, как PR, в котором пользователь числится ревьюером, закрывается, пользователь вновь доступен для назначения на ревью.

## Нереализованные идеи
- Назначение пользователей на ревью происходит случайным образом, однако чуть более справедливым будет случай, в котором у каждого пользователя в таблице хранится временная метка последнего ревью. В таком случае распределение на ревью будет происходить начиная с человека, проводившего ревью давнее всего. Эта идея не была реализована, дабы не отходить от заданной спецификации API.
- В таблице `users` на столбец `team_name` и в таблице `pull_requests` на столбец `author_id` имеет смысл добавить индекс для более быстрой фильтрации.
